atlassian

a brief overview

Billing core rewrite

Rewrote large portions of Atlassian's internal pricing and order processing monolith system. The new implementation allowed for a more flexible pricing configuration. This included pricing structure grandfathering, multiple contract term lengths and a multiple APIs and tools for marketplace vendors to manage product pricing.

This work laid the foundation for further projects like experimental pricing for customer subsets and usage based pricing.

Microservice architecture

Extracted several system components out of the core billing monolith application. Where possible these systems would run in parallel with the existing implementation in order to verify their behaviour and performance. This architecture helped our rapidly growing team by reducing the cost of onboarding new team member by reducing the domain knowledge required to work on a subset of functionality.

New systems are now created as a microservice by default, allowing teams to use the best tools for the job.

Continuous integration

During the growth from 8 to over 40 developers we had to drastically change the way we deployed our applications. We implemented a new functional test suite to cover the large legacy monolith system. 10 years of slapdash development had resulted in huge implementation detail unknowns.

The new end to end testing suite helped build deployment confidence. The build times for these services were drastically reduced, which provided quicker feedback for developers.

Microservices had to be deployable from master at any time, requiring a set comprehensive unit and functional test suites.
